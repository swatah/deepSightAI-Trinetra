# Glossary

Key terms and concepts used throughout the deepSightAI Trinetra documentation.

---

## A

**API (Application Programming Interface)**
- The RESTful HTTP interface that clients use to interact with deepSightAI Trinetra services.
- Base URL: `https://api.trinetra.com/v1`

**Audit Logging**
- Immutable recording of all user actions and system events for compliance and security monitoring.
- All audit entries stored in WORM (Write Once Read Many) storage with 7+ year retention.

**Authentication**
- The process of verifying user identity, typically via JWT tokens or API keys.
- See: [User Guide → Authentication](user-guide/auth.md)

**Authorization**
- Determining what actions a user can perform based on roles and permissions (RBAC).
- Implemented via middleware that checks JWT claims against required roles.

---

## C

**CLIP (Contrastive Language-Image Pre-Training)**
- The vision-language model used to generate embeddings for semantic search.
- OpenCLIP variant trained on large-scale image-text pairs.

**Container**
- Docker container encapsulating a single service (API, extractor, embedder, etc.).
- All services run as isolated containers in production.

**Cosine Similarity**
- Metric used to compare query embedding with frame embeddings (range 0-1).
- Higher scores indicate better semantic match.

---

## E

**Embedding**
- Numerical vector representation of an image (video frame) in multi-dimensional space.
- Generated by the embedder service using the CLIP model.
- Stored in Milvus vector database for similarity search.

**Extractor**
- Service responsible for video processing: segmenting videos and extracting frames.
- Runs as scalable worker pool; pulls jobs from Redis queue.

---

## F

**Frame**
- Single still image extracted from a video (typically 1 per second).
- Frames are stored temporarily in MinIO, then embedded and discarded.

---

## G

**GitOps**
- Deployment methodology using Git as the single source of truth for infrastructure.
- ArgoCD automatically syncs cluster state with Git repository.

---

## H

**HNSW (Hierarchical Navigable Small World)**
- Approximate nearest neighbor search algorithm used by Milvus.
- Provides sub-linear search complexity for high-dimensional vectors.

---

## K

**Kubernetes (K8s)**
- Container orchestration platform used for production deployments.
- deepSightAI Trinetra packages services as Helm charts for K8s.

**Kustomize**
- Kubernetes native configuration management tool used for environment overlays.
- Allows customization of base manifests without templating.

---

## M

**Milvus**
- Open-source vector database used to store and search frame embeddings.
- Handles billions of vectors with low-latency similarity search.

**MinIO**
- S3-compatible object storage used for videos, frames, and checkpoints.
- Provides high-performance object storage with erasure coding.

**mTLS (Mutual TLS)**
- Certificate-based authentication where both client and server verify each other.
- Used for service-to-service communication in secure deployments.

---

## O

**OpenAPI Specification**
- Standard for describing REST APIs (formerly Swagger).
- deepSightAI Trinetra provides OpenAPI specs for all services for client generation and documentation.

---

## P

**Partitioning (Multi-tenancy)**
- Data isolation strategy where each tenant's data is stored in separate partitions/collections.
- In Milvus: partitions named by `tenant_id`. In PostgreSQL: schemas or RLS.

**PostgreSQL**
- Relational database used for metadata storage (videos, segments, users, tenants).
- Also used for audit logs with WORM constraints.

**Prometheus**
- Metrics collection system used to monitor all services.
- Services expose `/metrics` endpoint in Prometheus text format.

---

## R

**Redis**
- In-memory data store used for service registry, job queues, and caching.
- Also used for pub/sub messaging between components.

**RBAC (Role-Based Access Control)**
- Authorization model where permissions are assigned to roles, and users assigned to roles.
- deepSightAI Trinetra supports roles: `admin`, `operator`, `viewer`.

---

## S

**Segment**
- 30-second chunk of a video used for parallel processing.
- Videos are split into segments for extractor parallelism.

**SIEM (Security Information and Event Management)**
- System that aggregates and analyzes security events.
- deepSightAI Trinetra audit logs forwarded to SIEM (Splunk, Elastic) via Kafka.

**Streamlit**
- Python-based web framework used for the user interface.
- Provides rapid prototyping with interactive components.

---

## T

**Tenant**
- An organization or customer instance in a multi-tenant deployment.
- All data isolated by `tenant_id`; tenants cannot access each other's data.

**TLS (Transport Layer Security)**
- Protocol for encrypting network communications.
- All external APIs require TLS 1.2+; internal service traffic uses mTLS.

---

## V

**Vector**
- See [Embedding](#embedding).

**Versioning (Documentation)**
- Using `mike` to maintain multiple versions of documentation side-by-side.
- Allows users toreference docs for specific releases (e.g., v1.0, v1.1, latest).

---

## W

**WORM (Write Once Read Many)**
- Storage characteristic that prevents modification or deletion of written data.
- Used for audit logs to ensure compliance with regulations requiring immutability.
